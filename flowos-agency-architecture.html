현재 운영 방식

FlowOS는 KJ(개발 리드), JP(개발자), Anton(클라이언트 운영)으로 구성된다. 클라이언트 인테이크는 항상 대면 미팅으로 시작한다 — 세 명 모두 참석하여 요구사항을 파악한다. 미팅 이후 KJ가 핵심 기술을 구현해 1–2일 내 MVP를 완성하고, JP가 코드 품질을 높여 프로덕션 수준으로 끌어올린다. Anton은 클라이언트와의 모든 외부 커뮤니케이션을 전담한다.

활성 프로젝트는 지속적으로 증가한다. 다수가 최근 2주 이내 커밋이 있고, 나머지는 유지보수 또는 MVP 대기 상태다. 수동 관리로는 이 규모를 유지할 수 없다.

왜 하나가 아니라 세 대인가

OpenClaw는 단일 사용자 게이트웨이다. 3명이 공유하면 컨텍스트 오염, 인증 범위 혼재, 단일 장애점이 발생한다. 3대의 Mac Mini — Antoni(코디네이터), Kyungjini(개발 리드), Jaepini(품질) — 는 각각 독립 에이전트를 운영하며, 하이브리드 코디네이터 토폴로지로 상호 감시·자동 페일오버한다.

핵심 원칙: 인원이 아닌 프로젝트 수에 비례해 확장되는 인프라. 다음 프로젝트의 한계 비용은 0에 수렴한다. 노드 하나가 죽어도 나머지가 자율 대응한다.

네트워크 구성 — Tailscale

Antoni · Kyungjini · Jaepini 세 노드는 Tailscale로 프라이빗 메시를 형성한다. 각 노드는 고정 Tailscale IP(100.64.0.x)와 hostname(*.tail)을 가지며, 공개 인터넷을 통하지 않고 노드 간 직접 통신한다. 모든 노드는 동일한 Google 계정(macmini@flowos.work)을 사용하며, Gmail 스캔은 코디네이터(기본: Antoni)만 수행한다.

Tailscale이 공유하는 것 vs 아닌 것: Tailscale은 네트워크 레이어만 제공한다. 파일이나 컨텍스트를 자동 동기화하지 않는다. 노드 간 공유 상태는 Supabase(프로젝트 데이터·node_roles·agent_events), GitHub(코드·CLAUDE.md), Slack(알림·메시지 버스)을 통해 이뤄진다. 에이전트 간 실시간 조율은 Supabase Realtime이 담당한다. 각 노드의 OpenClaw 로컬 메모리(~/.openclaw/memory/MEMORY.md)는 의도적으로 분리 유지된다.

git 전략

FlowOS는 main 직접 push 방식을 사용한다. PR은 실험적 기능 개발 시에만 예외적으로 사용한다. main push = 즉시 Vercel 프로덕션 배포이므로, 에이전트는 배포 실패를 실시간 감지하고 즉각 Slack 알림을 발송해야 한다. 스테이징 환경은 없다 — 이는 의도적 선택이며 속도를 최우선으로 한다.

프로젝트 레벨 락 — 동시 작업 방지

OpenClaw + Claude Code 통합

OpenClaw 에이전트는 판단하고, Claude Code CLI가 실제로 코드를 작성한다. API 호출이 아닌 실제 claude 바이너리 — subscription 과금.

역할 분리: OpenClaw = 두뇌 (데이터 수집 · 판단 · 프롬프트 구성). Claude Code CLI = 손 (파일 수정 · 빌드 · git push · 배포). 두 프로세스는 완전히 분리되며 비용도 독립적이다.

가장 큰 변화

Anton의 반복 업무 제거: 배포 완료 시 클라이언트에게 수동으로 카카오톡/이메일을 보내는 작업이 완전 자동화된다. 에이전트가 초안을 생성하고 Anton이 검토 후 발송하거나, 신뢰 수준이 높아지면 완전 자동 발송으로 전환할 수 있다.

Kyungjini의 컨텍스트 스위칭 비용 감소: 다수의 활성 프로젝트 사이를 오갈 때, 에이전트가 각 프로젝트의 마지막 상태, 미해결 이슈, 우선순위를 아침 브리핑으로 정리해 준다. Slack을 열면 오늘 뭘 해야 하는지 즉시 파악된다.

비용 가시성: 현재 15개 이상의 서비스에서 발생하는 API 비용이 어떤 프로젝트에 귀속되는지 파악하기 어렵다. 컨트롤 플레인이 프로젝트별로 비용을 집계하면 수익성 분석과 클라이언트 비용 청구가 정확해진다.

확장 비용 제로: 다음 프로젝트를 추가하는 비용은 `create-project` 한 번 실행뿐이다. 4번째 팀원이 합류하면 Mac Mini 한 대를 추가하면 된다. 마이그레이션 오버헤드 없음.

자율 복구: Antoni가 다운되면 Kyungjini가 30분 내 코디네이터를 자동 인수한다. Gmail 스캔, 일일 리포트, 비용 집계가 중단 없이 계속된다. 인간 개입 없이 시스템이 스스로 복구한다.

안전한 자율성: 킬 스위치, 서킷 브레이커, 확인 레벨 시스템으로 에이전트 폭주를 방지한다. 이메일 발송은 항상 사전 확인, 배포는 rate limit, 비용은 자동 캡 — 자율적이되 통제된 시스템.
